20:T4559,<h3 class="mt-5 mb-3"><strong>CORS: Cross-Origin Resource Sharing</strong></h3><p>웹 보안 정책: 동일 출처 정책(Same-Origin Policy, SOP)</p><p>다른 출처(origin)에서 실행되는 스크립트를 방지한다.</p><p>하지만 현실적으로 브라우저의 도메인과 백엔드 서버의 origin이 다른 경우는 흔하다. 이를 해결하기 위해 CORS(Cross-Origin Resource Sharing)가 도입되었다.</p><h2 class="mt-5 mb-3"><strong>CORS</strong></h2><h3 class="mt-5 mb-3"><strong>CORS의 역할</strong></h3><ul class="tight" data-tight="true"><li><p>동일 출처 정책을 우회하여 허용된 외부 도메인에서 API 요청을 가능하게 함.</p></li><li><p>안전한 방식으로 웹 애플리케이션이 타 도메인 리소스에 접근할 수 있도록 허용.</p></li><li><p>REST API 및 클라이언트-서버 구조에서 필수적인 보안 기능.</p></li></ul><p>사실 가장 중요한 점은, <strong>CORS 요청 제한은 브라우저에서 실행된다는 점이다.</strong> 오늘날의 프론트 개발 영역이 서버사이드와 클라이언트가 혼합된 상황이므로, CORS 정책을 그저 서버 개발자들이 해주어야 할 일로 떠넘기기엔 부족한 것 같다. (점차 서버의 코드를 더 많이 만질 일이 생기지 않을까도 생각이 들기에.)</p><p>CORS 설정은 서버에서 CORS 관련 헤더를 적절히 설정하여 특정 출처에서의 요청을 허용한다.</p><h2 class="mt-5 mb-3"><strong>CORS의 동작 방식</strong></h2><h3 class="mt-5 mb-3"><strong>Preflight 요청과 단순 요청</strong></h3><p>CORS 요청은 크게 단순 요청(Simple Request)과 사전 요청(Preflight Request)으로 나뉜다.</p><h4 class="mt-5 mb-3">1. 단순 요청 (Simple Request)</h4><ul class="tight" data-tight="true"><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">GET</code>, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">HEAD</code>, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">POST</code> 메서드만 사용 가능.</p></li><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Content-Type</code>은 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">application/x-www-form-urlencoded</code>, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">multipart/form-data</code>, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">text/plain</code>만 허용.</p></li><li><p>추가적인 CORS 검증 없이 즉시 서버에 요청을 보냄.</p></li></ul><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-http">GET /data HTTP/1.1
Host: api.example.com
Origin: https://example.com</code></pre><p>서버가 CORS를 허용한다면, 응답 헤더에 다음과 같이 포함된다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-http">Access-Control-Allow-Origin: https://example.com</code></pre><h4 class="mt-5 mb-3">2. Preflight 요청 (OPTIONS 요청)</h4><ul class="tight" data-tight="true"><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">PUT</code>, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">DELETE</code>, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">PATCH</code> 등의 메서드 사용.</p></li><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Authorization</code>, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Content-Type: application/json</code> 같은 커스텀 헤더 포함 시.</p></li><li><p>먼저 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">OPTIONS</code> 메서드로 서버에 해당 요청을 허용하는지 확인한 후 본 요청을 보냄.</p></li></ul><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-http">OPTIONS /data HTTP/1.1
Host: api.example.com
Origin: https://example.com
Access-Control-Request-Method: PUT
Access-Control-Request-Headers: Content-Type</code></pre><p>서버가 CORS를 허용하면 다음과 같이 응답한다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-http">HTTP/1.1 204 No Content
Access-Control-Allow-Origin: https://example.com
Access-Control-Allow-Methods: PUT, DELETE, PATCH
Access-Control-Allow-Headers: Content-Type
Access-Control-Max-Age: 86400</code></pre><h2 class="mt-5 mb-3"><strong>CORS 헤더 설정</strong></h2><p>CORS는 서버가 응답 헤더를 설정함으로써 적용된다. 주요 헤더는 다음과 같다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-http"> Access-Control-Allow-Origin: https://example.com | "*" 와일드 카드
    Access-Control-Allow-Methods: GET, POST, PUT, DELETE
    Access-Control-Allow-Headers: Content-Type, Authorization
    Access-Control-Allow-Credentials: true
    Access-Control-Max-Age: 86400l
    
  </code></pre><ol class="my-3 px-1 py-3 text-sm leading-6 bg-muted/30 border border-secondary rounded list-decimal tight" data-tight="true"><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Access-Control-Allow-Origin</code></p><ul class="tight" data-tight="true"><li><p>허용할 사용한다.</p></li><li><p>하지만, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">*</code>를 사용할 경우 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Authorization</code><strong> 헤더와 </strong><code class="mx-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">*</code>을 설정하면 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Credentials</code>가 활성화되지 않는다.</p></li></ul></li><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Access-Control-Max-Age</code>: Preflight 요청을 캐싱하여 불필요한 OPTIONS 요청을 줄인다.</p></li></ol><h2 class="mt-5 mb-3"><strong>CORS 정책 적용 방법</strong></h2><h3 class="mt-5 mb-3"><strong>서버</strong></h3><p>요청을 받고 응답을 보낸다. 주의해야할 점은, CORS에서 요청 헤더를 검증하는 것이다.</p><p>서버에서 CORS에 영향을 줄 수
