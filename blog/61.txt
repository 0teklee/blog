20:T9966,<h1 class="mt-5 mb-3">TLDR</h1><ul class="tight" data-tight="true"><li><p>그래프는 여러 노드가 엣지로 연결된 자료 구조이다.</p></li><li><p>JS에선 그래프를 인접 행렬과 인접 리스트로 만든다.</p></li></ul><hr class="my-5"><h1 class="mt-5 mb-3">정의</h1><blockquote><p>컴퓨터 과학에서 그래프는 수학의 그래프 이론에서 정의된 무방향 그래프와 방향 그래프 개념을 구현하기 위한 추상 자료형이다. - <a target="_blank" rel="noopener noreferrer nofollow" href="https://en.wikipedia.org/wiki/Graph_(abstract_data_type)">wikipedia (graph)</a></p></blockquote><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-mermaid">graph TD

A[A] --&gt; B[B];
A --&gt; C[C];
B --&gt; D;
B --&gt; E;
C --&gt; F;
C --&gt; G;</code></pre><p>이전 글에서 HTML 트리, DOM api의 자료 구조를 비교하며 "그래프"에 대해 조금은 더 근본적인 탐구를 해보았다. 문제 풀이에만 집중하다 보니, 기본 개념을 제대로 안살펴본 것 같아 그래프 구조, 탐색, 구체적 탐색법을 나눠 글을 작성했다. </p><p></p><p>작성하면서 느낀 거지만, 알면 알수록 깊고 심오한 내용으로 다가온다.(?)</p><hr class="my-5"><h2 class="mt-5 mb-3">기본 요소</h2><p>그래프의 기본 요소들은 다양한 이름으로 불리지만, 아래의 정의로 통일한다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>    A ---- 0번 노드 
  /   \ 
 B     C ---- 2번 노드
 └ 1번 노드
 
예시 노드 { 인덱스 : 저장 값 }
{ 0 : "A" } { 1 : "B" } { 2 : "C" }</code></pre><p><strong>노드</strong></p><ul class="tight" data-tight="true"><li><p>그래프를 구성하는 기본 단위로, <strong>데이터를 저장하는 개체</strong>이다.</p></li><li><p>노드 인덱스 + 노드 값 : 노드는 고유한 식별자와 안에 저장된 데이터로 구성된다.</p></li></ul><p><strong>노드 인덱스</strong>: 노드의 식별자. 0, 1, 2을 인덱스로 사용해 노드에 접근하고, 그래프 전체를 탐색할 수 있다.<br><strong>노드 데이터</strong>(값): 각 노드에 저장된 값. 노드는 인덱스와 별도의 데이터(값, 라벨, 가중치 등)를 저장할 수 있다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>노드 = ●    엣지: ● --- ● (노드 사이의 선)
높이 = 3

┌            ● - 루트         - 레이어 0
|           / \
|          /   \
높이       ●     ● - 깊이 = 1  - 레이어 1
|        / \   / \
|       ●   ● ●   ●          - 레이어 2
|      /           \
└     ●             ● - 리프  - 레이어 3</code></pre><ul class="tight" data-tight="true"><li><p><strong>루트와 리프</strong> : 트리 구조에서 최상위 노드를 루트, 자식이 없는 노드를 리프라 한다.</p></li><li><p><strong>엣지(간선)</strong>: 노드 간의 관계를 나타내며, 두 노드를 연결하는 선이다.</p></li><li><p><strong>깊이</strong>: 주로 트리 구조에서 사용된다. 깊이(depth)는 특정 노드에서 루트까지의 거리이다.</p></li><li><p><strong>레이어(층위)</strong>: 그래프에서 특정 노드가 몇 단계 깊이(depth)에 위치하는지를 나타낸다.</p></li><li><p><strong>높이</strong>: <strong>그래프 전체의 최대 깊이</strong>는 <strong>높이</strong>(height)라고 부른다. 예를 들어, 트리에서 높이는 루트 노드부터 가장 깊은 리프 노드까지의 길이로 정의한다.</p></li></ul><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>방향:                 
   ●        ● &lt;--&gt; ● &lt;--&gt; ●    
 ↙   ↘ 
●     ●


순환: 어느 노드에서 출발해도 재자리로 돌아올 수 있음
    ●
  ↗   ↘ 
 ●  ←  ● 

가중치: ● --(5)-- ●  (엣지에 숫자로 표현)</code></pre><ul class="tight" data-tight="true"><li><p><strong>방향</strong>: 엣지가 단방향인지, 양방향인지에 따라 방향 그래프와 무방향 그래프로 구분된다.</p></li><li><p><strong>순환</strong>: 그래프 내에서 동일한 노드로 돌아올 수 있는 경로가 있을 때 <strong>순환</strong>한다.</p></li><li><p><strong>가중치</strong>: <strong>엣지</strong>에 부여된 값으로, 특정 노드에서 다른 노드로 <strong>이동하는 비용이나 거리 등을 나타낸다</strong>.</p></li></ul><hr class="my-5"><h2 class="mt-5 mb-3">종류</h2><p>그래프는 가장 상위의 개념이다. 방향, 순환, 가중치 등 요소로 분류할 수 있다. 예를 들어 HTML 트리는 비순환 방향 비가중치 그래프이다. 어떤 그래프인 지 감이 잡히지 않을 때, 조금 더 구체적으로 그래프를 파악할 수 있다.</p><p><strong>순환</strong></p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>비순환 그래프       순환 그래프
     ●              ●
   ↙   ↘          ↗   ↘ 
  ●     ●        ●  ←  ●  
                  ↖   ↙
     0              ●              
   ↙   ↖          
  1  ←  2 
  비순환 그래프</code></pre><ul class="tight" data-tight="true"><li><p><strong>순환 그래프</strong> : 노드들이 특정 경로를 따라 다시 자기 자신으로 돌아올 수 있는 그래프.</p></li><li><p><strong>비순환 그래프</strong> : 순환 경로가 존재하지 않는 그래프.</p></li></ul><p><strong>방향</strong></p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>● --- ● --- ●     무방향 그래프
● --&gt; ● --&gt; ●     단방향 그래프
● &lt;--&gt; ● &lt;--&gt; ●   양방향 그래프</code></pre><ul class="tight" data-tight="true"><li><p><strong>방향 그래프</strong> : 엣지에 방향성이 존재하는 그래프. (단방향/양방향)</p></li><li><p><strong>무방향 그래프</strong> : 엣지에 방향성이 없는 그래프.</p></li></ul><p><strong>가중치</strong></p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>● --(5)-- ●  가중치 그래프
● ------- ●  비가중치 그래프</code></pre><ul class="tight" data-tight="true"><li><p><strong>가중치 그래프</strong> : 엣지에 가중치 값이 있는 그래프.</p></li><li><p><strong>비가중치 그래프</strong> : 엣지에 가중치가 없는 그래프</p></li></ul><h3 class="mt-5 mb-3">트리*</h3><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>이진 트리
	      A  --루트----┐       
	   ↙     ↘        |  
 	  B       C    ---┼- 서브 트리           
    ↙  ↘     ↙        | 
   D    E   F      ---┘   
   └ 리프 (D~F)   </code></pre><p>트리는 프론트엔드 개발에서 자주 사용되며, 특히 코딩 테스트에선 <strong>이진 트리</strong>가 자주 사용된다.</p><p><strong>특징</strong></p><ul class="tight" data-tight="true"><li><p><strong>단방향, 비순환 그래프</strong>이다.</p></li><li><p><strong>루트 노드는 하나이며</strong>, 이로 인해 <strong>레이어가 계층</strong>을 이룬다.</p></li><li><p>임의의 두 노드를 연결하는 <strong>유일한 경로만 존재</strong>한다.</p></li><li><p>일반적으로 루트에서 리프까지 내려갈 수록 노드의 숫자가 점차 증가한다.</p></li><li><p><strong>부분과 전체의 구조가 일치</strong></p></li><li><p><strong>서브 트리</strong>: 특정 노드를 루트로 하는 하위 트리. (ex: 위에선 A-B-C, B-D-E, C-F) </p><ul class="tight" data-tight="true"><li><p>서브 트리도 전체 트리처럼 여전히 트리이다 (ex: C-F).</p></li></ul></li><li><p><strong>이진 트리</strong> : 각 노드가 최대 두 개의 자식을 가지는 트리.</p></li></ul><p>이외로 다양한 형태의 트리가 있다.</p><ul class="tight" data-tight="true"><li><p><strong>힙</strong>: 특정 규칙에 따라 구성된 <strong>완전 이진 트리</strong>.</p></li><li><p><strong>AVL 트리</strong> : 자동 균형 조정이 이루어지는 <strong>이진 탐색 트리</strong>.</p></li><li><p><strong>레드-블랙 트리</strong> : 삽입, 삭제 시 균형을 유지하는 <strong>이진 탐색 트리</strong>.</p></li><li><p><strong>B-트리</strong> : 데이터베이스 시스템에서 널리 사용되는 <strong>균형 트리 구조</strong>.</p></li></ul><h3 class="mt-5 mb-3">힙</h3><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>힙
	      8  ---- 루트(최대 값)     
	   ↙     ↘          
 	  4       5              
    ↙  ↘     ↙  ↘          
   2    3   2    3 ---- 리프(최소 값)    </code></pre><p>힙은 <strong>노드에 저장된 값을 특정 규칙에 따라 구성한 이진 트리</strong>이다. 노드 데이터가 무엇이든, 엣지로 연결된 노드들을 그래프로 간주하지만, 힙은 <strong>노드 데이터에 따라 노드 트리로된다</strong>. "특정 규칙"은 결국 이렇게 표현될 수 있다.</p><ul class="tight" data-tight="true"><li><p><strong>최대 힙</strong> : 부모 노드가 자식보다 항상 큰 값을 가짐.</p></li><li><p><strong>최소 힙</strong> : 부모 노드가 자식보다 항상 작은 값을 가짐.</p></li></ul><p>힙은 특정 규칙으로 우선 순위를 반영할 수 있다. 새로운 개념이 많이 등장하니, 상세한 내용은 추후 살펴보도록 한다.</p><p><em>힙에 새로운 노드가 추가되거나 삭제되면, "</em><strong><em>힙 속성</em></strong><em>"을 유지하기 위한 로직이 수행된다. JS에서는 보통 배열로 구현되며, 우선순위 큐Priority Queue로 정렬한다.</em></p><h3 class="mt-5 mb-3">기타 유형</h3><p>이외에도 다양한 그래프가 있다.</p><ul class="tight" data-tight="true"><li><p><strong>멀티 그래프</strong>: 두 노드 사이에 여러 개의 엣지가 존재할 수 있는 그래프.</p></li><li><p><strong>유사 그래프</strong>: 멀티그래프의 한 유형으로, 자기 자신을 향하는 엣지를 포함할 수 있음.</p></li><li><p><strong>완전 그래프</strong>: 모든 노드가 서로 연결된 그래프.</p></li><li><p><strong>이분 그래프</strong>: 노드 집합이 두 개의 그룹으로 나뉘고, 같은 그룹 내에서는 엣지가 없는 그래프.</p></li><li><p><strong>평면 그래프</strong>: 엣지가 교차하지 않고 평면에 그릴 수 있는 그래프.</p></li><li><p><strong>희소 그래프</strong>: 엣지 개수가 노드 개수에 비해 상대적으로 적은 그래프.</p></li><li><p><strong>밀집 그래프</strong>: 엣지 개수가 노드 개수에 비해 상대적으로 많은 그래프.</p></li></ul><hr class="my-5"><h1 class="mt-5 mb-3">그래프 in JS</h1><h2 class="mt-5 mb-3">인접 행렬과 인접 리스트*</h2><p>대표적인 그래프 구현 방법은 <strong>인접 리스트와 인접 행렬</strong>이다. 목적에 따라 적합한 형태를 활용한다.</p><hr class="my-5"><h3 class="mt-5 mb-3">인접 행렬</h3><p>인접 행렬은 <strong>그래프의 노드 연결 관계를 2차원 배열</strong>(행렬)로 표현하는 방식이다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>인접 행렬의 구조
배열[노드1][노드2] = 값 (값으로 노드1,2의 연결 관계를 표현한다.)
배열[행 번호 row][열 번호 col] = 0 | 1 (not 연결/연결)
row와 col의 최댓값은 같아야한다.</code></pre><ul class="tight" data-tight="true"><li><p><strong>인접 행렬에서 노드 인덱스는 노드 데이터와 같다.</strong></p></li><li><p><strong>인접 행렬에서 노드 인덱스는 배열의 인덱스와 동일하다.</strong></p></li><li><p>두 노드의 연결 여부(엣지)가 인접 행렬에 저장된다.</p></li><li><p>2차원 배열(행렬)에서 <strong>노드 1은 행 번호</strong>, <strong>2는 열 번호</strong>이다.</p></li><li><p><strong>노드가 N개면 NxN 크기의 배열</strong>로 행렬을 나타낸다.</p></li></ul><h4 class="mt-5 mb-3">노드 데이터의 의미</h4><p>인접 행렬에서 두 노드를 통해 값을 얻을 수 있다. 값은 다양한 형태와 의미를 갖을 수 있다.</p><p>가장 기본적인 방식은 인접 행렬이 두 노드의 <strong>연결 관계를 나타내는 것</strong>이다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-js">const adjacencyMatrix = [
  [0, 1, 1, 0], // 0번 노드는 1, 2와 연결
  [1, 0, 0, 1], // 1, 2번 노드는 0, 3과 연결
  [1, 0, 0, 1],
  [0, 1, 1, 0] // 3번 노드는 1, 2와 연결
];

/* 
이 행렬을 그래프로 표현하면 다음과 같다
    0 
   / \
  1   2
   \ /
    3
 모두 연결된 순환 그래프인 것을 알 수 있다.
*/</code></pre><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">adjacencyMatrix</code>의 두 인덱스는 그래프에서 두 노드를 가리키고, <strong>데이터는 엣지</strong>를 나타낸다. <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">adjacencyMatrix</code>는 노드가 4개로 이루어진 그래프를 표현한다.</p><ul class="tight" data-tight="true"><li><p>0번 노드와 1번 노드는 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">adjacencyMatrix[0][1] = 1</code>이므로 연결되어있다.</p></li><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">adjacencyMatrix[0][3] = 0</code>으로 노드 0,3은 연결되어 있지 않다.</p></li></ul><h4 class="mt-5 mb-3">장단점</h4><p>✅ <strong>장점:</strong></p><ul class="tight" data-tight="true"><li><p>구현이 직관적이고 단순하다.</p></li><li><p>두 노드의 인덱스로 행렬에 O(1)로 접근해 값을 얻을 수 있다.</p></li></ul><p>✅ <strong>단점:</strong></p><ul class="tight" data-tight="true"><li><p>행렬이기 때문에 불필요한 공간 낭비가 생긴다.</p></li><li><p>가장 큰 노드 인덱스를 기준으로 행렬 크기가 결정되어 차이가 클 수록 공간 낭비가 커진다. </p><ul class="tight" data-tight="true"><li><p>3 노드의 인덱스가 1, 2, 999 이라면, <strong>가장 큰 인덱스 999로 전체 행렬이 만들어진다</strong></p></li></ul></li><li><p>이런 경우 행렬에는 연결되지 않음을 나타내는 불필요한 값이 행렬 대부분을 차지한다.</p></li><li><p>간선이 노드보다 매우 적은 그래프를 희소 그래프라고 하며, 이때 행렬은 대부분 비효율적이다.</p></li></ul><h4 class="mt-5 mb-3">NOTE</h4><p>❓ <strong>인접 행렬은 항상 두 노드의 연결 여부를 나타낼까</strong></p><p><strong>노드 인덱스</strong><br>인접 행렬에서 인덱스가 곧 노드인 경우가 일반적이지만, 예외도 많다.</p><ul class="tight" data-tight="true"><li><p><strong>변환(예: 문자열 ID, 객체)</strong>: 데이터를 <strong>인접 행렬로 변환</strong>해야 할 때. 데이터와 행렬을 맵핑하면서 노드 인덱스를 숫자로 만들고, 행렬에 두 노드 연결 여부를 반영해야 한다.</p></li><li><p><strong>노드 번호가 특정 범위(예: 1부터 시작)인 경우</strong>: <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">matrix[1][1]</code>부터 사용하며, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">0</code>번 인덱스는 비워둘 수 있다.</p></li></ul><p><strong>노드 데이터 (배열의 값)</strong><br>인접 행렬에 들어있는 <strong>값</strong>은 <strong>문제에 따라 다를 수 있다</strong>. 조건에 따라 노드 값은 연결 관계를 나타낼 수도, 아닐 수도 있다. (ex: 노드 값 = 가중치 ) 혹은 <strong>노드 연결 관계를 구현하는 것</strong> 자체가 문제일 수도 있다.</p><p>경험 상 <strong>DFS, BFS 등 그래프 탐색</strong>에 사용해야 하는 경우가 잦아 인접 행렬의 기본 개념을 확실히 해두는 것이 중요하다.</p><hr class="my-5"><h3 class="mt-5 mb-3">인접 리스트</h3><p>인접 리스트는 반대다. 인접 행렬과 달리, <strong>연결된 노드만 저장하여 공간 효율성이높다</strong>. 인접 리스트는 보통 <strong>배열</strong>로 <strong>그래프 연결 관계</strong>를 표현한다.</p><p><strong>인접 리스트 배열에서 인덱스는 보통 노드 인덱스를 가리킨다</strong>.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-js">인접 리스트 예시
인덱스 = 노드 번호 
배열[인덱스] = 값 // 값은 연결 관계인 노드 번호나 가중치 등 다양한 형태가 될 수 있다.</code></pre><p>하지만 인접 행렬과 마찬가지로 주어진 문제에 따라 다양하게 표현되거나 변환이 필요할 수 있다.</p><h4 class="mt-5 mb-3">노드 데이터의 의미</h4><p>배열로 표현된 인접리스트는 다음과 같다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-js">// 1차원 배열
const adjacencyList = [-1, 0, 0, 1, 1, 2, 2, 3]
// 1,2번 노드는 0번과, 3,4번 노드는 1번과, 5,6은 2, 7은 3과 연결
// 보통 루트는 -1로 표기하나,
const adjacencyList = [Infinity, 0, 1, 1, 2, 2, 3, 3]
// 인덱스 0번에 빈 값으로 Infinity 등 빈값을 넣기도 한다.

// 저장 값이 더 복잡한 형태로 표현된 인접 리스트
const weightedGraph = [
  [[1, 5], [2, 10]], // 노드 0이 노드 1과 가중치 5, 노드 2와 가중치 10으로 연결됨
  [[0, 5], [2, 3]],  // 노드 1이 노드 0과 5, 노드 2와 3으로 연결됨
  [[0, 10], [1, 3]]  // 노드 2가 노드 0과 10, 노드 1과 3으로 연결됨
];</code></pre><h4 class="mt-5 mb-3">장단점</h4><p>장점과 단점은 인접 행렬과 반대이다.</p><p>✅ <strong>장점:</strong></p><ul class="tight" data-tight="true"><li><p>공간 효율성이 좋다. 메모리를 적게 사용한다.</p></li><li><p>인접 행렬과 반대로 희소 그래프(노드가 엣지보다 상대적으로 많은 경우) 유리하다.</p></li><li><p>그래프 표현이 인접 행렬보다 직관적이다. 불필요한 값이 줄어들고, 연결 관계와 노드 값만 남긴다.</p></li></ul><p>✅ <strong>단점:</strong></p><ul class="tight" data-tight="true"><li><p>인접 행렬보다 <strong>탐색 효율이 낮다</strong>. </p><ul class="tight" data-tight="true"><li><p>최악의 경우 모든 엣지를 확인해야 하여 <strong>O(E = 엣지 갯수)</strong>이 될 수 있다.</p></li></ul></li><li><p>그래프가 가독성이 떨어진다. 데이터 구조가 복잡해질 수록 더 파악이 어려울 수 있다. (위의 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">weightedGraph</code> 가 예시이다.)</p></li></ul><h4 class="mt-5 mb-3">NOTE</h4><p>❓ <strong>인접 리스트의 인덱스와 값은 항상 노드 인덱스와 엣지(연결 관계)일까</strong></p><ul class="tight" data-tight="true"><li><p>노드 인덱스와 노드에 저장된 값을 적절히 설계해야 한다.</p></li></ul><p>배열을 사용해 인덱스를 노드로, 값을 엣지로 하는 것은 직관적이다. 하지만 행렬과 마찬가지로 <strong>인접 리스트를 직접 만드는 것 자체가 문제 의도일 수 있다.</strong></p><p><strong>노드 인덱스</strong></p><ul class="tight" data-tight="true"><li><p>그래프 목적과 자료 구조에 따라 인덱스가 항상 노드 번호를 나타내지 않을 수 있다.</p></li></ul><p>배열 인덱스로 엣지를 나타낼 수 없는 경우도 흔하다.</p><ul class="tight" data-tight="true"><li><p><strong>노드의 고유한 ID가 문자열</strong>이거나, 특정 데이터 구조가 필요한 경우 </p><ul class="tight" data-tight="true"><li><p>노드는 배열 인덱스가 아닌 <strong>키가 노드</strong>를 식별하고, <strong>값이 연결 정보</strong>를 저장하게 된다.</p></li><li><p>배열 대신 객체(Map)를 활용하여 노드의 식별자를 키로 사용하는 방식도 가능하다.</p></li></ul></li></ul><p><strong>노드 데이터 (배열 요소)</strong><br><strong>노드에 저장된 값이 추가적인 정보</strong>를 포함하는 경우<br>- 노드 값이 단순히 다음 노드를 가리키는 것이 아니라 가중치나 방문 여부 등을 나타낼 수도 있다.</p><p>인접 리스트가 주어지면, <strong>인덱스와 배열 내부의 값이 각각 어떤 의미인지</strong> 파악하는 것이 중요한 단서를 준다. 이를 토대로 추론을 이어가야 수월하다.</p><p>경험 상 코딩 테스트 문제에서 그래프는 인덱스로 연결된 두 배열을 인접 행렬이나 인접 리스트로 만들어야 하는 경우가 많았다. ex) <em>arr1과 arr2는 인덱스로 연결된 두 (ex: 위에선 A-B-C, B-D-E, C-F) </p><ul class="tight" data-tight="true"><li><p>서브 트리도 전체 트리처럼 여전히 트리이다 (ex: C-F).</p></li></ul></li><li><p><strong>이진 트리</strong> : 각 노드가 최대 두 개의 자식을 가지는 트리.</p></li></ul><p>이외로 다양한 형태의 트리가 있다.</p><ul class="tight" data-tight="true"><li><p><strong>힙</strong>: 특정 규칙에 따라 구성된 <strong>완전 이진 트리</strong>.</p></li><li><p><strong>AVL 트리</strong> : 자동 균형 조정이 이루어지는 <strong>이진 탐색 트리</strong>.</p></li><li><p><strong>레드-블랙 트리</strong> : 삽입, 삭제 시 균형을 유지하는 <strong>이진 탐색 트리</strong>.</p></li><li><p><strong>B-트리</strong> : 데이터베이스 시스템에서 널리 사용되는 <strong>균형 트리 구조</strong>.</p></li></ul><h3 class="mt-5 mb-3" id="-4">힙</h3><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="hljs language-lua">힙
	      <span class="hljs-number">8</span>  <span class="hljs-comment">---- 루트(최대 값)     </span>
	   ↙     ↘          
 	  <span class="hljs-number">4</span>       <span class="hljs-number">5</span>              
    ↙  ↘     ↙  ↘          
   <span class="hljs-number">2</span>    <span class="hljs-number">3</span>   <span class="hljs-number">2</span>    <span class="hljs-number">3</span> <span class="hljs-comment">---- 리프(최소 값)    </span></code></pre><p>힙은 <strong>노드에 저장된 값을 특정 규칙에 따라 구성한 이진 트리</strong>이다. 노드 데이터가 무엇이든, 엣지로 연결된 노드들을 그래프로 간주하지만, 힙은 <strong>노드 데이터에 따라 노드 트리로된다</strong>. "특정 규칙"은 결국 이렇게 표현될 수 있다.</p><ul class="tight" data-tight="true"><li><p><strong>최대 힙</strong> : 부모 노드가 자식보다 항상 큰 값을 가짐.</p></li><li><p><strong>최소 힙</strong> : 부모 노드가 자식보다 항상 작은 값을 가짐.</p></li></ul><p>힙은 특정 규칙으로 우선 순위를 반영할 수 있다. 새로운 개념이 많이 등장하니, 상세한 내용은 추후 살펴보도록 한다.</p><p><em>힙에 새로운 노드가 추가되거나 삭제되면, "</em><strong><em>힙 속성</em></strong><em>"을 유지하기 위한 로직이 수행된다. JS에서는 보통 배열로 구현되며, 우선순위 큐Priority Queue로 정렬한다.</em></p><h3 class="mt-5 mb-3" id="-5">기타 유형</h3><p>이외에도 다양한 그래프가 있다.</p><ul class="tight" data-tight="true"><li><p><strong>멀티 그래프</strong>: 두 노드 사이에 여러 개의 엣지가 존재할 수 있는 그래프.</p></li><li><p><strong>유사 그래프</strong>: 멀티그래프의 한 유형으로, 자기 자신을 향하는 엣지를 포함할 수 있음.</p></li><li><p><strong>완전 그래프</strong>: 모든 노드가 서로 연결된 그래프.</p></li><li><p><strong>이분 그래프</strong>: 노드 집합이 두 개의 그룹으로 나뉘고, 같은 그룹 내에서는 엣지가 없는 그래프.</p></li><li><p><strong>평면 그래프</strong>: 엣지가 교차하지 않고 평면에 그릴 수 있는 그래프.</p></li><li><p><strong>희소 그래프</strong>: 엣지 개수가 노드 개수에 비해 상대적으로 적은 그래프.</p></li><li><p><strong>밀집 그래프</strong>: 엣지 개수가 노드 개수에 비해 상대적으로 많은 그래프.</p></li></ul><hr class="my-5"><h1 class="mt-5 mb-3" id="in-js">그래프 in JS</h1><h2 class="mt-5 mb-3" id="-6">인접 행렬과 인접 리스트*</h2><p>대표적인 그래프 구현 방법은 <strong>인접 리스트와 인접 행렬</strong>이다. 목적에 따라 적합한 형태를 활용한다.</p><hr class="my-5"><h3 class="mt-5 mb-3" id="-7">인접 행렬</h3><p>인접 행렬은 <strong>그래프의 노드 연결 관계를 2차원 배열</strong>(행렬)로 표현하는 방식이다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="hljs language-scss">인접 행렬의 구조
배열<span class="hljs-selector-attr">[노드1]</span><span class="hljs-selector-attr">[노드2]</span> = 값 (값으로 노드<span class="hljs-number">1</span>,<span class="hljs-number">2</span>의 연결 관계를 표현한다.)
배열<span class="hljs-selector-attr">[행 번호 row]</span><span class="hljs-selector-attr">[열 번호 col]</span> = <span class="hljs-number">0</span> | <span class="hljs-number">1</span> (not 연결/연결)
row와 col의 최댓값은 같아야한다.</code></pre><ul class="tight" data-tight="true"><li><p><strong>인접 행렬에서 노드 인덱스는 노드 데이터와 같다.</strong></p></li><li><p><strong>인접 행렬에서 노드 인덱스는 배열의 인덱스와 동일하다.</strong></p></li><li><p>두 노드의 연결 여부(엣지)가 인접 행렬에 저장된다.</p></li><li><p>2차원 배열(행렬)에서 <strong>노드 1은 행 번호</strong>, <strong>2는 열 번호</strong>이다.</p></li><li><p><strong>노드가 N개면 NxN 크기의 배열</strong>로 행렬을 나타낸다.</p></li></ul><h4 class="mt-5 mb-3">노드 데이터의 의미</h4><p>인접 행렬에서 두 노드를 통해 값을 얻을 수 있다. 값은 다양한 형태와 의미를 갖을 수 있다.</p><p>가장 기본적인 방식은 인접 행렬이 두 노드의 <strong>연결 관계를 나타내는 것</strong>이다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="hljs language-js"><span class="hljs-keyword">const</span> adjacencyMatrix = [
  [<span class="hljs-number">0</span>, <span class="hljs-number">1</span>, <span class="hljs-number">1</span>, <span class="hljs-number">0</span>], <span class="hljs-comment">// 0번 노드는 1, 2와 연결</span>
  [<span class="hljs-number">1</span>, <span class="hljs-number">0</span>, <span class="hljs-number">0</span>, <span class="hljs-number">1</span>], <span class="hljs-comment">// 1, 2번 노드는 0, 3과 연결</span>
  [<span class="hljs-number">1</span>, <span class="hljs-number">0</span>, <span class="hljs-number">0</span>, <span class="hljs-number">1</span>],
  [<span class="hljs-number">0</span>, <span class="hljs-number">1</span>, <span class="hljs-number">1</span>, <span class="hljs-number">0</span>] <span class="hljs-comment">// 3번 노드는 1, 2와 연결</span>
];

<span class="hljs-comment">/*

    0 
   / \
  1   2
   \ /
    3
 모두 연결된 순환 그래프인 것을 알 수 있다.

인덱스는 그래프에서 두 노드를 가리키고, <strong>데이터는 엣지</strong>를 나타낸다. <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">adjacencyMatrix</code>는 노드가 4개로 이루어진 그래프를 표현한다.</p><ul class="tight" data-tight="true"><li><p>0번 노드와 1번 노드는 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded <strong>장점:</strong></p><ul class="tight" data-tight="true"><li><p>구현이 직관적이고 단순하다.</p></li><li><p>두 노드의 인덱스로 행렬에 O(1)로 접근해 값을 얻을 수 있다.</p></li></ul><p>✅ <strong>단점:</strong></p><ul class="tight" data-tight="true"><li><p>행렬이기 때문에 불필요한 공간 낭비가 생긴다.</p></li><li><p>가장 큰 노드 인덱스를 기준으로 행렬 크기가 결정되어 차이가 클 수록 공간 낭비가 커진다. </p><ul class="tight" data-tight="true"><li><p>3 노드의 인덱스가 1, 2, 999 이라면, <strong>가장 큰 인덱스 999로 전체 행렬이 만들어진다</strong></p></li></ul></li><li><p>이런 경우 행렬에는 연결되지 않음을 나타내는 불필요한 값이 행렬 대부분을 차지한다.</p></li><li><p>간선이 노드보다 매우 적은 그래프를 희소 그래프라고 하며, 이때 행렬은 대부분 비효율적이다.</p></li></ul><h4 class="mt-5 mb-3">NOTE</h4><p>❓ <strong>인접 행렬은 항상 두 노드의 연결 여부를 나타낼까</strong></p><p><strong>노드 인덱스</strong><br>인접 행렬에서 인덱스가 곧 노드인 경우가 일반적이지만, 예외도 많다.</p><ul class="tight" data-tight="true"><li><p><strong>변환(예: 문자열 ID, 객체)</strong>: 데이터를 <strong>인접 행렬로 변환</strong>해야 할 때. 데이터와 행렬을 맵핑하면서 노드 인덱스를 숫자로 만들고, 행렬에 두 노드 연결 여부를 반영해야 한다.</p></li><li><p><strong>노드 번호가 특정 범위(예: 1부터 시작)인 경우</strong>: <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">matrix[1][1]</code>부터 사용하며, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">0</code>번 인덱스는 비워둘 수 있다.</p></li></ul><p><strong>노드 데이터 (배열의 값)</strong><br>인접 행렬에 들어있는 <strong>값</strong>은 <strong>문제에 따라 다를 수 있다</strong>. 조건에 따라 노드 값은 연결 관계를 나타낼 수도, 아닐 수도 있다. (ex: 노드 값 = 가중치 ) 혹은 <strong>노드 연결 관계를 구현하는 것</strong> 자체가 문제일 수도 있다.</p><p>경험 상 <strong>DFS, BFS 등 그래프 탐색</strong>에 사용해야 하는 경우가 잦아 인접 행렬의 기본 개념을 확실히 해두는 것이 중요하다.</p><hr class="my-5"><h3 class="mt-5 mb-3" id="-8">인접 리스트</h3><p>인접 리스트는 반대다. 인접 행렬과 달리, <strong>연결된 노드만 저장하여 공간 효율성이높다</strong>. 인접 리스트는 보통 <strong>배열</strong>로 <strong>그래프 연결 관계</strong>를 표현한다.</p><p><strong>인접 리스트 배열에서 인덱스는 보통 노드 인덱스를 가리킨다</strong>.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="hljs language-js">인접 리스트 예시
인덱스 = 노드 번호 
배열[인덱스] = 값 <span class="hljs-comment">// 값은 연결 관계인 노드 번호나 가중치 등 다양한 형태가 될 수 있다.</span></code></pre><p>하지만 인접 행렬과 마찬가지로 주어진 문제에 따라 다양하게 표현되거나 변환이 필요할 수 있다.</p><h4 class="mt-5 mb-3">노드 데이터의 의미</h4><p>배열로 표현된 인접리스트는 다음과 같다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="hljs language-js"><span class="hljs-comment">// 1차원 배열</span>

];</code></pre><h4 class="mt-5 mb-3">장단점</h4><p>장점과 단점은 인접 행렬과 반대이다.</p><p>✅ <strong>장점:</strong></p><ul class="tight" data-tight="true"><li><p>공간 효율성이 좋다. 메모리를 적게 사용한다.</p></li><li><p>인접 행렬과 반대로 희소 그래프(노드가 엣지보다 상대적으로 많은 경우) 유리하다.</p></li><li><p>그래프 표현이 인접 행렬보다 직관적이다. 불필요한 값이 줄어들고, 연결 관계와 노드 값만 남긴다.</p></li></ul><p>✅ <strong>단점:</strong></p><ul class="tight" data-tight="true"><li><p>인접 행렬보다 <strong>탐색 효율이 낮다</strong>. </p><ul class="tight" data-tight="true"><li><p>최악의 경우 모든 엣지를 확인해야 하여 <strong>O(E = 엣지 갯수)</strong>이 될 수 있다.</p></li></ul></li><li><p>그래프가 가독성이 떨어진다. 데이터 구조가 복잡해질 수록 더 파악이 어려울 수 있다. (위의 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">weightedGraph</code> 가 예시이다.)</p></li></ul><h4 class="mt-5 mb-3">NOTE</h4><p>❓ <strong>인접 리스트의 인덱스와 값은 항상 노드 인덱스와 엣지(연결 관계)일까</strong></p><ul class="tight" data-tight="true"><li><p>노드 인덱스와 노드에 저장된 값을 적절히 설계해야 한다.</p></li></ul><p>배열을 사용해 인덱스를 노드로,
