20:T75dc,<blockquote><p>TLDR: 키가 필터링을 수행할 수 있도록 키-값 저장 구조를 만든다</p></blockquote><p>이번 글에서는 프론트엔드 알고리즘 테스트에 자주 활용되는 해시를 알아본다. 기본 개념과 JS에서 활용하는 방법을 위주로 살펴본다. 이 글은 책을 통해 알게 된 개념과 스스로 학습하는 과정을 기록했다. 특히 JS부터 추론 과정을 글로 표현하는 것에 집중했다.</p><h2 class="mt-5 mb-3">Hash?</h2><h3 class="mt-5 mb-3">무엇?</h3><p>키-값value으로 저장하는 자료 구조이다.<br>키와 값이 일대일로 대응한다.</p><h3 class="mt-5 mb-3">어떻게?</h3><p>기본 원리는 동일하다. 키를 해시 함수로 변환하여 인덱스를 얻는다.<br>저장 공간 해당 인덱스에 값을 저장한다.<br>해시 함수 구현 방법에 따라 키-&gt;인덱스를 만드는 방법이 달라진다.</p><h3 class="mt-5 mb-3">왜?</h3><p>핵심은 <strong>효과적인 탐색</strong>이다. 이유는:</p> <p>- 키만 알면 값을 바로 얻을 수 있다.</p><p>- 0~N의 숫자 인덱스보다 해시 키가 인지하기 쉽다.</p><h3 class="mt-5 mb-3">언제?</h3><p>데이터를 <strong>키-값 구조로</strong> 만들어 효과적인 <strong>탐색</strong>할 때.<br>예를 들어 탐색을 O(n)에서 O(1)로 만들 수 있을 때.<br>실생활에서는 해시 함수의 단방향성으로 보안에 활용된다.</p><hr class="my-5"><h2 class="mt-5 mb-3">특징</h2><p>간략히 특징을 정리하자면,</p><ol class="my-3 px-1 py-3 text-sm leading-6 bg-muted/30 border border-secondary rounded list-decimal tight" data-tight="true"><li><p>키를 인덱스로 활용하려면 적절한 변환 과정(=해시 함수)가 필요하다.</p></li><li><p>단방향이다. 값을 통해선 키를 찾을 수는 없다. 오로지 '키'를 해시 함수에 넣어야 값을 얻을 수 있다.</p></li><li><p>키 자체가 인덱스로 기능하여, O(1)로 값을 얻을 수 있다.</p></li></ol><h3 class="mt-5 mb-3">해시 함수 종류</h3><p>해시 함수를 구성하는 여러 전략이 있다. 책에 언급된 모듈러 연산(%)을 중심으로 알아본다.</p><ol class="my-3 px-1 py-3 text-sm leading-6 bg-muted/30 border border-secondary rounded list-decimal tight" data-tight="true"><li><p>나누기 : h(x) = x mod K </p><ul class="tight" data-tight="true"><li><p>값 x를 소수 K로 모듈러 연산한다.</p></li><li><p>x의 인덱스는 0~K-1까지 갖을 수 있다.</p></li><li><p>이 때 소수 K가 해시 전체 크기가 된다. 즉 크기 제한이 생긴다.</p></li></ul></li><li><p>곱하기 : h(x) = (((x <em> A) mod 1) </em> M) </p><ul class="tight" data-tight="true"><li><p>나누기의 크기 한계를 개선한다.</p></li><li><p>값 x에 황금비 A를 곱한다</p></li><li><p>-&gt; 이후 모듈러 연산으로 1을 제외한 소수 자리만 얻은 뒤</p></li><li><p>-&gt; 이 수를 테이블 크기 M에 곱한다.</p></li><li><p>이때 인덱스는 0~M-1에 배치된다.</p></li></ul></li><li><p>문자열 해싱 </p><ul class="tight" data-tight="true"><li><p>키 x가 문자열일 때도 해싱을 적용한다.</p></li><li><p>각 문자를 숫자로 변환하는 해시함수를 사용한다.<br>(이 부분은 더 깊은 내용을 다룬다. 그러나 주제에서 벗어나므로, 원문 참조.)</p></li></ul></li></ol><h3 class="mt-5 mb-3">충돌 문제</h3><p>해시 함수의 문제 상황</p><ol class="my-3 px-1 py-3 text-sm leading-6 bg-muted/30 border border-secondary rounded list-decimal tight" data-tight="true"><li><p>해시 함수가 동일한 키(인덱스)를 반환할 때 충돌이 생길 수 있다.</p></li><li><p>값이 저장되는 해시의 크기 제한이 있다.</p></li></ol><p>모두 저장하려는 키-값이 손상될 수 있는 충돌 문제를 갖고 있다.<br>문제 해결 위해 다음과 같은 방법들이 제시된다.</p><ul class="tight" data-tight="true"><li><p>체이닝: 중복된 인덱스가 생성됐을 때, 그 인덱스에 새로운 값을 다음 노드로 추가한다. (연결 리스트) </p><ul class="tight" data-tight="true"><li><p>탐색 성능 저하 : 중복된 인덱스에서 키에 해당하는 값을 찾을 때까지 연결 노드의 next를 확인해서 O(n)이 된다.</p></li><li><p>공간 활용성 저하 : 해시에 빈 공간이 있어도 활용하지 못하고 같은 인덱스에 값을 저장한다.</p></li></ul></li><li><p>개방 주소법: 충돌 시 빈 공간에 값을 저장한다. </p><ul class="tight" data-tight="true"><li><p>선형 탐사 : 충돌 시 특정 간격으로 이동하며 빈 공간을 찾아 저장한다. 이 때 간격 배치로 인해 값이 특정 위치에 몰리는 클러스터가 발생할 수 있다.</p></li><li><p><strong>이중 해싱</strong> : <strong>해싱 함수를 2개 작성</strong>한다. 첫 함수는 값을 키로 변환하고, 두번째는 충돌 발생 시 이동하는 로직으로 구성한다.</p></li></ul></li></ul><hr class="my-5"><h2 class="mt-5 mb-3">JS에서 해시</h2><p>JS 알고리즘 코딩 테스트라는 맥락에선 해시의 효율적인 탐색에 집중한다. key-value의 일대일 대응으로 가능한 O(1) 탐색이 해시 사용의 목적이다. 이어지는맥락에서 '해시'는 셋 모두를 가리킨다. (유사한 구조임에는 인지)</p><p>JS는 해시와 유사한 자료구조를 제공한다. 바로 객체(Object)와 Set, Map이다. 그럼 이 세가지 중에서 어떻게 적절한 사용 케이스를 구분할까?</p><h3 class="mt-5 mb-3">JS 유사 해시 객체 특징</h3><p><strong>일반 객체(Object)</strong></p><ul class="tight" data-tight="true"><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">add(x)</code>, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">delete(x)</code> 로 수정할 수 있으며, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">values()</code>, <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">keys()</code> 등을 활용해 순회할 수 있다.</p></li><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono 이전 값을 덮어쓰기 한다.</p></li><li><p><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">.get(key)</code> 를 사용해 target이 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">nums[i] + nums[j]</code> 이 같을 때 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">[i, j]</code>를 반환한다.</p></li><li><p>조건 3: 원본 배열 nums의 인덱스를 유지해야 한다.</p></li><li><p>조건 4: nums는 반드시 target을 만들 수 있는 요소를 포함한다. 즉 i, j는 반드시 있다. 예외처리 해야할 부분이 줄어든다.</p></li></ul><p>이를 바탕으로  문제를 이해하기 쉽게 바꿔보면,</p><p>O(n)으로 순회하면서 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">nums[i] + nums[j] = target</code>인 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">[i, j]</code>를 반환하시오.</p><h3 class="mt-5 mb-3" id="-11">모델 추론하기</h3><p>이제 i , j 를 찾는 가장 적당한 도구를 찾아보자. 이 도구들을 앞으로 일종의 사고 모델로 가정한다.</p><p>해시는 '효율적인 탐색'을 위한 모델이다. 키만 있으면 O(1)로 값을 바로 얻을 수 있다. 지금 상황에 맞아 떨어지는 것 같다. 해시를 풀이 과정에 적용해보면,</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="hljs language-mermaid">graph TD
    A["A. O(n)으로 nums를 순회"] --> B["B. 현재 값은 nums[i] (i = 인덱스)"]
    B --> C["C. 해시에서 나머지 한 쌍 nums[j] 탐색 (O(1))"]
    C -->|찾음| D["결과 반환: [i, j]"]
    C -->|못 찾음| E["현재 값 nums[i]를 해시에 저장"]

  <p>적절해 보인다. 하지만 어떻게? 구현을 하려니 여러 고민이 생긴다.</p><ol class="my-3 px-1 py-3 text-sm leading-6 bg-muted/30 border border-secondary rounded list-decimal"><li><p>해시로 어떤 객체를 쓰지?</p></li><li><p>어떻게 해시에 저장하지?</p></li><li><p>어떻게 해시에서 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">nums[i]</code>를 찾지?</p></li><li><p>해시에 적당한 객체 찾기<br><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Object {} ||  new Set() || new Map()</code><br>문제 조건에 맞는 적당한 객체를 고민하는게 생각보다 많은 단서를 준다.</p></li></ol><p>그래도 감이 안잡히면 이렇게 좁혀간다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="hljs language-mermaid">graph TD;
