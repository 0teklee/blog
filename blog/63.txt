class="mt-5 mb-3">1. 기본 개념</h1><h2 class="mt-5 mb-3">1-1. 방문과 조회</h2><p>DFS(깊이 우선 탐색) 알고리즘은 목적에 따라 <strong>방문과 조회 순서를 결정</strong>해야 한다. 이에 따라서 구체적인 함수 구성이 크게 달라질 수 있다.</p><p></p><p><strong>방문(Visit):</strong> 새로운 노드로 이동하는 과정</p><ul class="tight" data-tight="true"><li><p>대부분의 경우 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">visited</code>라는 변수로 방문 여부를 기록해야 한다.</p></li><li><p><strong>방문 여부 확인과 업데이트는 전체 로직에 중요한 영향을 끼친다.</strong></p></li><li><p>방문 순서 = 순회 방식은 주어진 문제마다 달라질 수 있다.</p></li><li><p>목적에 따라 방문과 조회가 함께 이루어지거나, 그렇지 않을 수 있다.</p></li></ul><p><strong>조회(Query):</strong> 현재 노드의 데이터를 확인하는 과정</p><ul class="tight" data-tight="true"><li><p>대부분의 경우 조건에 따라 노드 데이터를 판별해야 한다.</p></li><li><p>노드 데이터를 기반으로 데이터를 가공할 때가 많다.</p></li><li><p>ex) 노드 데이터가 1인 노드들의 연결 관계, 경로, 노드 데이터를 바탕의 탐색 등</p></li></ul><p><strong>핵심은 방문과 조회의 순서를 주어진 문제에 맞게 구성하는 것이다.</strong> 하지만 매번 다른 모습의 DFS 함수를 떠올리기는 쉽지 않다. 그래서 <strong>기본 원리</strong>를 제대로 알아야 한다.</p><p></p><p>DFS는 <strong>가장 깊이 있는 노드까지 탐색</strong>한 후, <strong>더 이상 탐색할 노드가 없으면 백트래킹(되돌아가기)한다</strong>. 모든 DFS는 <strong>탐색 순서의 메커니즘을 스택</strong><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Stack</code><strong>으로 구현</strong>한다.</p><p></p><p>JS에서 스택은 기본적이다.</p><ol class="my-3 px-1 py-3 text-sm leading-6 bg-muted/30 border border-secondary rounded list-decimal tight" data-tight="true"><li><p>콜 스택(재귀 함수): JS에서 함수 실행 순서는 스택으로 관리된다. 가장 나중에 호출된 함수부터 실행된다.</p></li><li><p>명시적인 스택 : 배열로 스택을 만들 수 있다. push, pop 메서드로 스택을 쉽게 관리할 수 있다.</p></li></ol><p>재귀 함수를 사용하느냐, 스택을 명시하느냐에 따라 방문과 조회의 순서를 조정할 수 있다. 일반적으로 <strong>재귀 함수는 전위 순회</strong>를 따른다.</p><h2 class="mt-5 mb-3">1-2. 백트래킹과 순회</h2><p><strong>백트래킹(Backtracking):</strong> 탐색을 진행하다가 더 이상 탐색할 노드가 없으면 이전 노드로 되돌아가는 과정<br><strong>순회(Traversal):</strong> 그래프의 노드를 일정한 순서로 방문하는 것. 그래프 전체의 탐색 순서 방식과 연관된다.</p><p></p><p>두 개념은 방문과 조회, 더 구체적으로 <strong>둘이 이루어지는 타이밍</strong>과 연관이 깊다. 이 부분에서 각 DFS 함수의 형태가 결정되기 때문이다.</p><p></p><p>DFS는 재귀 함수와 배열 스택으로 구현할 수 있다. 코드의 가독성, 성능 등에 다소 차이가 생길 수 있으나 적절한 형태를 선택하면 함수 구성이 쉬워진다.</p><ul class="tight" data-tight="true"><li><p>재귀 함수: 전위 순회를 기본으로 한다. 백트래킹, 순회 방식은 모두 '가장 깊은 노드'의 서브 트리를 따라간다.</p></li><li><p>배열 스택(명시적 스택): 함수 내부에 스택 변수를 만들고, pop, push 메서드로 순서를 관리한다. 스택에 먼저 추가된 노드가 나중에 실행된다.</p></li></ul><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code>함수 콜스택 쌓이는 것 = 방문이 먼저 일어난 순서대로

       - !graph[node]는 해당 노드가 리프 노드임을 의미.

        - 예를 들어, 8 → 4 → 2로 이동하면 path = [8, 4, 2], sum = 14가 됨.

    // 인접 노드 방문 이후에 백트래킹을 추가한다
	// 백트래킹: 방문 기록 삭제하면 다른 경로에서 다시 방문 가능하다
	visited.delete(node); 
}</code></pre><hr class="my-5"><h2 class="mt-5 mb-3">3-4. 순회(Traversal)</h2><p><strong><mark class="px-0.5 py-0.5/2 rounded text-white font-medium hover:animate-pulse" data-color="#108f3e" style="background-color: #108f3e; color: inherit">DFS는 많은 경우 전위 순회</mark></strong>이다. 앞서 살핀 구현체들은 기본적으로 DFS를 전위 순회로 구현한다. 전위 순회는 모든 노드를 탐색하여 데이터를 수집할 때 사용한다.</p><p>하지만 경우에 따라 중위, 후위 순회가 필요할 때도 있다. 이전 글에서 전/중/후위에 대한 내용을 다뤘으니, 구체적으로 DFS에서 중위, 후위 순회를 구현하는 방법을 알아보자.</p><h3 class="mt-5 mb-3">A. 중위 순회(Inorder Traversal)</h3><p>중위 순회는 "왼쪽 → 현재 노드 → 오른쪽" 순으로 방문한다. 이진 탐색 트리(BST)에서 정렬된 순서로 데이터를 얻고 싶을 때 사용한다. 하지만 드물게 주어진 그래프가 이진 탐색 트리가 아닐 때도 있다. 이런 경우 특정 기준에 따라 그래프 중심을 기준으로 왼쪽과 오른쪽을 나누어 탐색한다. (하지만 대부분 이진 탐색 트리가 주어진다.)</p><ul class="tight" data-tight="true"><li><p>이진 탐색 트리(BST)에서 정렬된 순서로 데이터를 얻고 싶을 때 </p><ul class="tight" data-tight="true"><li><p>ex: 이진 탐색 트리의 노드를 오름차순으로 출력</p></li></ul></li><li><p>그래프에서 특정 기준을 중심으로 왼쪽과 오른쪽을 나누어 탐색할 때</p></li></ul><p><strong>이진 탐색 트리 중위 순회하기</strong> :<br>이진 탐색 트리는 특정 규칙을 만족하는 계층 구조를 이룬다. 중위 순회에서 이진 탐색 트리와 함께 K번째 숫자가 무엇인지 혹은 숫자 N의 K가 무엇인지 묻는 질문이 일반적이다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="language-js">// 탐색하려는 트리가 이진 탐색 트리로 이미 구성된 경우이다.
// K번째 원소 찾기

let result = -1 // 못 찾을 시 -1 반환
let count = 0

function dfsInorderKthSmallest(tree, index, count, k) {
    if (index &gt;= tree.length) return; // 1. 배열 범위를 벗어나면 종료

    const leftIndex = 2 * index + 1; // 2. 왼쪽 자식 인덱스 계산
    const rightIndex = 2 * index + 2; // 3. 오른쪽 자식 인덱스 계산

    // 4. 왼쪽 서브트리 탐색 (BST에서 작은 값들이 위치)
    dfsInorderKthSmallest(tree, leftIndex, count, k, result);
    
    // 5. 현재 노드 방문 (중위 순회의 핵심: 왼쪽 탐색 후 현재 노드 방문)
    // 카운트를 더한다 = 노드 깊이가 깊어진다 (재귀 함수 호출될 때마다)
    count += 1;
    if (count === k) {

        result = tree[index]; 
        return; // 탐색을 종료한다
    }

    // 6. 오른쪽 서브트리 탐색 (BST에서 큰 값들이 위치)
    dfsInorderKthSmallest(tree, rightIndex, count, k, result);
}

function dfsInorderGraph(graph, index) {
// 1. 배열 범위를 벗어나거나 이미 방문한 노드라면 탐색 중단
    if (index &gt;= graph.length || visited.has(index)) return; 
    
    visited.add(index); // 2. 현재 노드를 방문 처리

    // 3. 인접 노드를 특정 기준(예: 숫자 크기순)으로 정렬
    let neighbors = graph[index] ? [...graph[index]]
    // 문제에 따라 정렬 기준이 달라진다 (숫자 크기순, 문자열 정렬, 가중치 등)
    .sort((a, b) =&gt; a - b)  
    : [];
    // 그래프[인덱스]가 인접 노드(인덱스)를 배열로 갖고 있어야

    let mid = Math.floor(neighbors.length / 2);
    // `mid`를 기준으로 왼쪽과 오른쪽을 나누어 탐색할 수 있다.
    // 정렬되지 않은 그래프에서 "왼쪽/오른쪽" 개념이 없기 때문에,
    // "중앙값"으로 왼쪽/오른쪽 기준을 정한다.

    // 4. 왼쪽 부분 탐색 (중앙 값 이전, 왼쪽 서브 트리를 먼저 탐색)
    for (let i = 0; i &lt; mid; i++) {
        dfsInorderGraph(graph, neighbors[i]);
    }

    // 5. 중위 순회 핵심 부분: 왼쪽을 탐색한 후, 현재 노드 방문
    result.push(index);
    // 현재 노드를 result에 저장하고, 오른쪽 서브 트리를 탐색한다.
    
    // 6. 오른쪽 부분 탐색 (오른쪽 서브 트리)
    for (let i = mid; i &lt; neighbors.length; i++) {
        dfsInorderGraph(graph, neighbors[i]);
    }

    if (visited.has(node)) return;

class="mt-5 mb-3" id="1">1. 기본 개념</h1><h2 class="mt-5 mb-3" id="1-1">1-1. 방문과 조회</h2><p>DFS(깊이 우선 탐색) 알고리즘은 목적에 따라 <strong>방문과 조회 순서를 결정</strong>해야 한다. 이에 따라서 구체적인 함수 구성이 크게 달라질 수 있다.</p><p></p><p><strong>방문(Visit):</strong> 새로운 노드로 이동하는 과정</p><ul class="tight" data-tight="true"><li><p>대부분의 경우 <code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">visited</code>라는 변수로 방문 여부를 기록해야 한다.</p></li><li><p><strong>방문 여부 확인과 업데이트는 전체 로직에 중요한 영향을 끼친다.</strong></p></li><li><p>방문 순서 = 순회 방식은 주어진 문제마다 달라질 수 있다.</p></li><li><p>목적에 따라 방문과 조회가 함께 이루어지거나, 그렇지 않을 수 있다.</p></li></ul><p><strong>조회(Query):</strong> 현재 노드의 데이터를 확인하는 과정</p><ul class="tight" data-tight="true"><li><p>대부분의 경우 조건에 따라 노드 데이터를 판별해야 한다.</p></li><li><p>노드 데이터를 기반으로 데이터를 가공할 때가 많다.</p></li><li><p>ex) 노드 데이터가 1인 노드들의 연결 관계, 경로, 노드 데이터를 바탕의 탐색 등</p></li></ul><p><strong>핵심은 방문과 조회의 순서를 주어진 문제에 맞게 구성하는 것이다.</strong> 하지만 매번 다른 모습의 DFS 함수를 떠올리기는 쉽지 않다. 그래서 <strong>기본 원리</strong>를 제대로 알아야 한다.</p><p></p><p>DFS는 <strong>가장 깊이 있는 노드까지 탐색</strong>한 후, <strong>더 이상 탐색할 노드가 없으면 백트래킹(되돌아가기)한다</strong>. 모든 DFS는 <strong>탐색 순서의 메커니즘을 스택</strong><code class="mx-1 px-1 bg-muted dark:bg-muted-foreground rounded text-orange-700 dark:text-theme italic font-mono transition-colors duration-300">Stack</code><strong>으로 구현</strong>한다.</p><p></p><p>JS에서 스택은 기본적이다.</p><ol class="my-3 px-1 py-3 text-sm leading-6 bg-muted/30 border border-secondary rounded list-decimal tight" data-tight="true"><li><p>콜 스택(재귀 함수): JS에서 함수 실행 순서는 스택으로 관리된다. 가장 나중에 호출된 함수부터 실행된다.</p></li><li><p>명시적인 스택 : 배열로 스택을 만들 수 있다. push, pop 메서드로 스택을 쉽게 관리할 수 있다.</p></li></ol><p>재귀 함수를 사용하느냐, 스택을 명시하느냐에 따라 방문과 조회의 순서를 조정할 수 있다. 일반적으로 <strong>재귀 함수는 전위 순회</strong>를 따른다.</p><h2 class="mt-5 mb-3" id="1-2">1-2. 백트래킹과 순회</h2><p><strong>백트래킹(Backtracking):</strong> 탐색을 진행하다가 더 이상 탐색할 노드가 없으면 이전 노드로 되돌아가는 과정<br><strong>순회(Traversal):</strong> 그래프의 노드를 일정한 순서로 방문하는 것. 그래프 전체의 탐색 순서 방식과 연관된다.</p><p></p><p>두 개념은 방문과 조회, 더 구체적으로 <strong>둘이 이루어지는 타이밍</strong>과 연관이 깊다. 이 부분에서 각 DFS 함수의 형태가 결정되기 때문이다.</p><p></p><p>DFS는 재귀 함수와 배열 스택으로 구현할 수 있다. 코드의 가독성, 성능 등에 다소 차이가 생길 수 있으나 적절한 형태를 선택하면 함수 구성이 쉬워진다.</p><ul class="tight" data-tight="true"><li><p>재귀 함수: 전위 순회를 기본으로 한다. 백트래킹, 순회 방식은 모두 '가장 깊은 노드'의 서브 트리를 따라간다.</p></li><li><p>배열 스택(명시적 스택): 함수 내부에 스택 변수를 만들고, pop, push 메서드로 순서를 관리한다. 스택에 먼저 추가된 노드가 나중에 실행된다.</p></li></ul><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="hljs language-css">함수 콜스택 쌓이는 것 = 방문이 먼저 일어난 순서대로
<span class="hljs-selector-tag">A</span> > <span class="hljs-selector-tag">B</span> > C 순서대로 (재귀) 함수 호출이 콜 스택에 쌓인다
			|__C__|
			|__B__|
			|__A__|

			|_____|
			|__B__|
C 실행		|__A__|

    }
    <span class="hljs-comment">// 재귀함수에서는 다음 노드로 이동하기 전에 현재 노드의 방문 여부를 업데이트 해야한다.</span>
    visited.<span class="hljs-title function_">add</span>(node); <span class="hljs-comment">// ✅ 올바른 타이밍: 인접 노드 방문 전에 업데이트 해야한다</span>

    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> neighbor <span class="hljs-keyword">of</span> graph[node]) {
    <span class="hljs-comment">// 콜 스택에 다음 노드를 추가한다</span>
        <span class="hljs-title function_">dfs</span>(graph, neighbor, visited); <span class="hljs-comment">// neighbor는 현재 노드 node와 연결돤 노드들이다.</span>
        <span class="hljs-comment">// graph[node]에 정렬된 순서대로 콜 스택에 쌓인다.</span>
        <span class="hljs-comment">// 가장 마지막에 추가된 노드가 가장 먼저 실행된다.</span>
    }

}</code></pre><p><strong>명시적인 스택</strong>: 반면에 배열 스택과 반복문은 방문 여부를 DFS 함수 내부에서도 선언할 수 있다.</p><pre class="ql-syntax hljs rounded-md p-4 my-2 text-sm"><code class="hljs language-js"><span class="hljs-comment">// 명시적인 스택 함수:</span>
<span class="hljs-keyword">function</span> <span class="hljs-title function_">stackDfs</span>(<span class="hljs-params">graph, start</span>) {
	<span class="hljs-keyword">const</span> visited = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Set</span>(); <span class="hljs-comment">// 방문 여부를 기록하는 Set</span>
